## Codebase Patterns
- The `Metric` interface in `pkg/metrics/metric.go` uses an accumulator pattern: `flush(timestamp float64, out []*Serie) ([]*Serie, error)`. All metric types append to `out` and return it.
- Use `GetSerie()` / `PutSerie()` from `pkg/metrics/serie_pool.go` for pooled *Serie allocation in flush paths.
- Test files for metrics (`pkg/metrics/*_test.go`) call `flush()` directly (same package). Any interface change requires updating these test callsites.
- Pre-existing test failure: `TestCheckMetrics` and `TestHistogramPercentiles` in `pkg/metrics` panic due to `SetWithoutSource` being called outside a proper test-scoped config mock. Not caused by our changes.
- Aggregator tests use `-tags test` build tag: `go test -tags test ./pkg/aggregator/...`
- `BenchmarkTimeSamplerFlushCardinality` benchmark creates a fresh sampler each iteration with `b.StopTimer()` during setup, so pool warmup doesn't happen within a single iteration. Alloc savings come from structural changes (eliminating wrapper slices) rather than pool hits.
- **sketchMap is now a struct** (not map type alias): use `newSketchMap()` constructor, not `make(sketchMap)`. Methods have pointer receivers.
- **sync.Pool get/put pattern for slices**: `putBinList(a []bin)` with `&a` causes a slice header escape-to-heap (1 alloc per call). Fix by having get return `([]T, *[]T)` and put take `*[]T` — reuses the existing heap-allocated header rather than creating a new one.
- **Agent.FinishAndReset()**: preferred over `Finish()` for pool-reuse paths — retains Buf/CountBuf backing arrays so next insert on a recycled agent avoids reallocation.
- **Pool hit timing**: benchmarks with `b.StopTimer()` setup show pool warmup across iterations (pool is package-level). Timer-ON allocs are ONLY from the flush phase. Pre-warm analysis: after ~10 cycles, backing arrays (bins, Buf) are retained via Reset, so subsequent Insert/flush cycles are 2 allocs/agent (Sketch.Copy).
