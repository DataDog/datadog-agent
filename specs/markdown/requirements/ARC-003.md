# `ARC-003` Hook-based package lifecycle
{bdg-ref-primary}`lifecycle <lifecycle>`
{bdg-ref-primary}`hooks <hooks>`

Package-specific behavior during install, remove, and upgrade operations is implemented
via a Hooks interface.

Each package type (datadog-agent, datadog-apm-inject, etc.) provides its own hook
implementations that handle platform-specific tasks such as service management,
file permissions, and SELinux context setup.

The hook interface includes:
- `PreInstall` / `PostInstall`
- `PreRemove`
- `PreStartExperiment` / `PostStartExperiment`
- `PreStopExperiment` / `PostStopExperiment`
- `PrePromoteExperiment` / `PostPromoteExperiment`
- `PostStartConfigExperiment` / `PreStopConfigExperiment` / `PostPromoteConfigExperiment`
- `PreInstallExtension` / `PostInstallExtension` / `PreRemoveExtension`
