# `FUN-014` Custom OCI registry and authentication
{bdg-ref-primary}`config <config>`
{bdg-ref-primary}`oci <oci>`
{bdg-ref-primary}`registry <registry>`

The operator shall be able to override the default OCI registry URL and choose an
authentication method for downloading packages.

Supported authentication methods:
- "docker": Use the Docker credential configuration (default).
- "gcr": Use Google Container Registry authentication.
- "password": Use explicit username/password credentials.

Registry and authentication overrides shall be configurable both globally and per-image,
allowing different packages to be pulled from different registries.
