# `NFR-004` Atomic repository operations
{bdg-ref-primary}`resilience <resilience>`
{bdg-ref-primary}`repository <repository>`

All repository operations (install, upgrade, experiment start/stop/promote) shall be atomic.

The on-disk repository state shall never be left in an invalid intermediate state,
even if an error occurs mid-operation. Garbage (unreferenced version directories)
may be left behind, but it shall be cleaned up by the next operation or by an
explicit garbage collection.
