# `FUN-025` User account management
{bdg-ref-primary}`windows <windows>`
{bdg-ref-primary}`msi <msi>`
{bdg-ref-primary}`user_account <user_account>`
> The installer shall support the following user account formats:
> user, .\user, netbiosname\user, verylonghostname\user, hostname.fqdn\user,
> domain\user, domain.fqdn\user, LocalSystem, SYSTEM, NT AUTHORITY\SYSTEM.
> 

The Windows installer shall support multiple user account formats for the Agent service user:
- `user` and `.\user`: Local account on workgroup hosts; domain account on domain controllers.
- `netbiosname\user`, `hostname.fqdn\user`: Explicit local accounts (error on domain controllers).
- `domain\user`, `domain.fqdn\user`: Domain accounts.
- `LocalSystem`, `SYSTEM`, `NT AUTHORITY\SYSTEM`: Built-in system accounts.

On domain controllers, ambiguous formats (`user`, `.\user`) shall refer to domain accounts,
and machine-local formats shall produce an error.

The installer shall create local accounts that do not exist (on workgroup hosts and domain clients).
The installer shall not create domain accounts; it shall error if the specified domain account
does not exist.

When a password is not provided for an existing local account, the installer shall reset the
password to a random value. The installer shall not change domain account passwords.

The installer shall never delete the `ddagentuser` account.
