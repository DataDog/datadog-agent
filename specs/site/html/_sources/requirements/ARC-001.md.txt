# `ARC-001` OCI-based package distribution
{bdg-ref-primary}`oci <oci>`
{bdg-ref-primary}`distribution <distribution>`

Packages are distributed as OCI container images and pulled from OCI-compliant registries
using the go-containerregistry library.

This architecture decision enables:
- Leveraging existing container registry infrastructure for package hosting.
- Supporting multiple authentication methods (Docker config, GCR, username/password).
- Using standard OCI annotations for package metadata.
- Efficient layer-based caching and transfer.
