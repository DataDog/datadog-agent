# `DEF-003` Experiment
{bdg-ref-primary}`experiment <experiment>`

An Experiment is a staged upgrade mechanism for OCI packages.

An experiment installs a new version of a package alongside the current stable version,
without removing the stable version's files. This allows the operator to test the new version
and either promote it to stable or roll back to the previous version.

The experiment lifecycle consists of three phases:
1. **Start**: The new version is installed and the experiment symlink is set.
2. **Promote**: The experiment becomes the new stable; the old stable is removed.
3. **Stop**: The experiment is discarded; the old stable remains unchanged.
