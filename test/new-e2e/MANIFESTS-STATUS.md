## Usage

### Generate manifests from CI config
```bash
# Generate for all areas that have CI jobs
dda e2e generate-manifests --dry-run

# Generate for specific area
dda e2e generate-manifests --area containers

# Regenerate existing manifest
dda e2e generate-manifests --area npm --force
```

### Validate manifests against CI
```bash
# Validate all manifests
dda e2e generate-ci-deps --validate

# Validate specific area
dda e2e generate-ci-deps --area containers --validate
```

## Workflow

**CI is the source of truth.** Manifests document what's configured in `.gitlab/test/e2e/e2e.yml`.

1. **Modify CI** - Update `.gitlab/test/e2e/e2e.yml` with needed artifacts
2. **Generate manifest** - `dda e2e generate-manifests --area myarea --force`
3. **Validate** - `dda e2e generate-ci-deps --validate` (or let pre-commit do it)
4. **Commit both** - CI config and manifest together

## Next Steps

## Manifest Guidelines

Manifests are **generated from CI**, not manually created:
- ✅ Generated by `dda e2e generate-manifests`
- ✅ Document current CI artifact dependencies
- ✅ Include test-specific patterns automatically detected
- ✅ Have comments showing source jobs
- ❌ Don't edit manually - regenerate instead
