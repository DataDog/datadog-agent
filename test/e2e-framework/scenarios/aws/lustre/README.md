## 1. What this scenario does

Provisions a single x86_64 EL9 EC2 host and bootstraps an **all-in-one Lustre
2.15** parallel filesystem on it (MGS/MGT + MDS/MDT + OSS/OST + client, all on
one node, over loopback LNet `0@lo`). It then runs a continuous I/O + metadata
workload against the client mount and installs the Datadog Agent with a
three-instance `lustre.d` check config so the released `lustre` integration emits
metrics for the `client`, `mds`, and `oss` node types.

- **Technology / version:** Lustre 2.15 (target point release >= 2.15.5 so the
  check's preferred `lctl ... -y` YAML path works).
- **Agent check:** `lustre` (integrations-core, **released**, min Agent 7.69.0).
  Namespace `lustre.*`. No service checks, no events.
- **Deployment shape:** `single-vm-all-in-one`. This is the upstream
  `llmount.sh` developer pattern (loop-device backing files over loopback LNet),
  not a downgrade — Docker Compose is impossible for Lustre because it needs
  out-of-tree kernel modules and a kernel-matched, Lustre-patched `e2fsprogs`
  (`mkfs.lustre`).
- **Interface:** CLI/host-kernel only. The check shells out to allowlisted
  absolute binaries `/usr/sbin/lctl`, `/usr/sbin/lnetctl`, `/usr/bin/lfs`
  (most with `sudo`). There is **no network listener** the Agent connects to;
  LNet TCP 988 is host-local on a single node.
- **Auth:** `auth_mode: none`. No license, no token, no integration credentials.
  The only secret in play is `DD_API_KEY`, injected automatically by the E2E
  framework's Agent install (`env.AgentAPIKey()`); it is never referenced in any
  generated file.

## 4. Generation profile and strategy

Interfaces: `cli-command`, `host-kernel`, `distributed-filesystem`
(storage-cluster also applies). No REST/SOAP/SNMP/network-listener.

Deployment feasibility:
- `docker_compose: impossible` — out-of-tree kmods + kernel-matched ldiskfs +
  `mkfs.lustre`. No faithful container exists, so **no `docker-compose.yaml` was
  generated** (output-contract rule 6 — no fake Compose).
- `single_vm: possible` — **chosen mode**. All Lustre roles co-locate on one
  EC2 host. Every metric *family* is produced on one node.
- `multi_vm: optional` — adds peer-NID realism; **not required** for metric
  coverage. (The local-integration pass rated multi-VM `required` for *full*
  coverage; reconciled to single-VM because all families are produced on one
  host, with `lustre.net.peer.*` and similar staying sparse — documented, not
  blocking.)

Hard constraints:
- `requires_kernel_modules: true` (lustre, lnet, ldiskfs/osd-ldiskfs, mgs, mdt,
  ost, obdfilter, ptlrpc).
- `requires_privileged_host: true` — root for `modprobe`/`losetup`/
  `mkfs.lustre`/`mount -t lustre`; `dd-agent` needs passwordless sudo for the
  three CLI binaries (granted by the generated sudoers drop-in).
- `requires_license: false`, `requires_credentials: false`,
  `requires_nested_virtualization: false`.
- Storage: 30-40 GB gp3 root (holds loop backing files for MGT/MDT/OST +
  Whamcloud RPMs + DKMS build artifacts); loop devices over sparse files.
- Network: default VPC, security group = SSH 22 inbound + all egress.
- Arch: **x86_64 only** (Whamcloud el9 *server* RPMs are x86_64; arm64 servers
  unsupported). OS: Rocky/RHEL 9 (EL9), kernel pinned to the EL minor matching
  the chosen 2.15.x build. Instance >= 16 GB RAM (m5.xlarge). SELinux permissive
  recommended.

Generation strategy: `create_agent_e2e_component`, `create_pulumi_infra`,
`create_host_setup_scripts`, `create_load_scripts` = **true**;
`create_compose`, `create_mock_or_simulator`, `create_fixture_replay`,
`create_external_lab_manifest` = **false**. `no_component_package: false` — the
component package is appropriate and honestly models setup + asset copy +
workload (this is **not** a last-resort no-component case). Fixture-replay
(`tests/fixtures/*.txt`) remains a CI-only fallback that was intentionally not
generated.

## 9. Assumptions and known risks

- **Highest risk — kernel <-> EL-minor <-> Whamcloud RPM coupling.** The running
  kernel must match the EL minor the Lustre kmod/DKMS RPMs were built against, or
  modules fail to load and `mkfs.lustre` fails. `install.sh` resolves the el9
  build and pins the kernel (`versionlock` + dnf `exclude=kernel*`) to mitigate,
  and DKMS rebuilds for the running kernel, but this is **unverified on real AWS
  this run** (`deploy: false`).
- **Rocky 9 AMI may be unwired.** `os.RockyLinux` exists but has no default
  descriptor/AMI in the framework. The scenario uses `os.RedHat9` (verified
  wired) — same dnf-family EL9 bootstrap. Reach Rocky only via explicit
  `ec2.WithAMI` / `ddinfra:osImageID`.
- **Two-phase reboot.** `install.sh` may reboot into the pinned kernel; the
  component orders `configure.sh` strictly after install via `PulumiDependsOn`
  and blocks on SSH reachability post-reboot. Scripts are idempotent so a human
  can re-run them while iterating on a live host.
- **node_type autodetect picks `mds`.** On an all-in-one host autodetection
  returns only `mds` (MDT present). The lab therefore configures **three
  explicit instances** — if you edit `conf.yaml` down to one, client/oss
  families are silently skipped.
- **jobstats / changelogs are conditional.** `lustre.job_stats.*` needs
  `jobid_var=procname_uid` set and a running job (the workload provides one);
  changelog logs additionally need `logs_enabled: true` (set by `WithLogs()`),
  `enable_changelogs: true`, and a registered changelog user. Validate these
  manually.
- **Expected-sparse on single-node loopback (not a defect):**
  `lustre.net.peer.*`, error/drop/timeout `*_count` counters, jobstats `.bucket`
  histograms on small instances, HSM ops, `mgs.exports.tgtreg.count`.
- **SELinux Enforcing untested.** Recommended permissive during the lab
  (`install.sh` drops it to permissive); Enforcing may block Lustre mounts or
  the dd-agent sudo CLIs.
- **5 s check timeout.** The check times out subprocess calls at 5 s; size the
  host generously (>= 16 GB RAM, m5.xlarge) so metrics are not dropped.
- **Wiki flags corroborated, not verified.** Exact `mkfs.lustre`/`lnetctl` flag
  syntax should be sanity-checked against the lustre.org wiki when reachable.
