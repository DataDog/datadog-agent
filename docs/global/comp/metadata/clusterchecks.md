> **TL;DR:** Collects and periodically sends a metadata payload describing the dispatch state of Cluster Checks (which node each check is assigned to, dangling counts) from the leader cluster agent instance to the Datadog inventory backend.

# comp/metadata/clusterchecks

**Team:** container-platform
**Import path:** `github.com/DataDog/datadog-agent/comp/metadata/clusterchecks`
**Build tag:** `clusterchecks` (both `impl/` and `fx/` are guarded by this tag)

## Purpose

The `clusterchecks` metadata component collects and periodically sends a metadata payload that describes the state of Cluster Checks to the Datadog inventory backend. It captures, for each dispatched or dangling check config: the check name, config hash, provider, source, init/instance configuration (scrubbed), and which node the check is assigned to. Summary counters for dangling checks and available nodes are also included.

The payload is only generated by the **leader** cluster agent instance. It is submitted to the backend when both `inventories_checks_configuration_enabled` is `true` and cluster identification (cluster name or cluster ID) is available. The payload is also available at the `/metadata/cluster-checks` HTTP endpoint and included in flares (`cluster-checks-metadata.json`).

The `clusterHandler` dependency is injected at runtime via `SetClusterHandler`, after Autodiscovery and the cluster checks dispatcher have been initialised.

## Key elements

### Key interfaces

```go
type Component interface {
    // WritePayloadAsJSON writes the scrubbed payload as JSON to an HTTP response.
    WritePayloadAsJSON(w http.ResponseWriter, r *http.Request)

    // SetClusterHandler injects the cluster checks dispatcher handler.
    // Must be called after the dispatcher is ready (post-Autodiscovery init).
    SetClusterHandler(handler *pkgclusterchecks.Handler)
}
```

### Key types

#### Payload

```go
type Payload struct {
    Clustername          string                 `json:"clustername"`
    ClusterID            string                 `json:"cluster_id"`
    Timestamp            int64                  `json:"timestamp"`
    ClusterCheckMetadata map[string][]metadata  `json:"clustercheck_metadata"`
    ClusterCheckStatus   map[string]interface{} `json:"clustercheck_status,omitempty"`
    UUID                 string                 `json:"uuid"`
}
```

`ClusterCheckMetadata` is keyed by check name. Each entry is a list of metadata maps (one per dispatched instance) containing:
- `config.hash`, `config.provider`, `config.source`
- `init_config`, `instance_config` — scrubbed YAML
- `node_name` — node the check is dispatched to (empty for dangling checks)
- `status` — `"DISPATCHED"` or `"DANGLING"`
- `errors` — non-empty for dangling checks

`ClusterCheckStatus` contains:
- `dangling_count` — number of checks not assigned to any node
- `node_count` — number of nodes available for dispatch

## Usage

### Registration

The component is wired in the cluster-agent start commands:

```
cmd/cluster-agent/subcommands/start/command.go
cmd/cluster-agent-cloudfoundry/subcommands/run/command.go
```

```go
clusterchecksmetadatafx "github.com/DataDog/datadog-agent/comp/metadata/clusterchecks/fx"
// ...
clusterchecksmetadatafx.Module()
```

### Injecting the handler at runtime

After the cluster checks Autodiscovery is set up, the dispatcher `Handler` is provided to the component:

```go
// cmd/cluster-agent/subcommands/start/command.go
clusterChecksMetadataComp.SetClusterHandler(clusterCheckHandler)
```

Without a handler, the component skips payload generation until one is set.

### HTTP endpoint

```
cmd/cluster-agent/api/server.go  — StartServer(..., clusterChecksMetadataComp clusterchecksmetadata.Component, ...)
cmd/cluster-agent/api/agent/agent.go
```

```go
r.HandleFunc("/metadata/cluster-checks", clusterChecksMetadataComp.WritePayloadAsJSON).Methods("GET")
```
