# Package `pkg/serverless`

## Purpose

`pkg/serverless` provides the building blocks for the Datadog serverless agent,
the in-process extension that runs alongside AWS Lambda functions (and other
serverless runtimes such as Google Cloud Run and Azure App Services/Container
Apps). It collects logs, custom metrics (via DogStatsD), and APM traces
produced by the function, buffers them during invocation, and synchronously
flushes them to the Datadog backend at the end of each invocation — before the
Lambda runtime freezes the execution environment.

The package is split into focused sub-packages; the root package exposes only
the `FlushableAgent` interface that ties them together.

### Architecture overview

```
Lambda function (user code)
        |
        v
  DogStatsD UDP (port 8125)      Lambda Telemetry / Logs API
        |                                  |
        v                                  v
pkg/serverless/metrics             pkg/serverless/logs
ServerlessMetricAgent              SetupLogAgent / SetLogsTags
(DogStatsD + Demultiplexer)        (logs pipeline over a channel)
        |                                  |
        v                                  v
        └──────── Datadog backend ─────────┘
                        ^
                        |
             pkg/serverless/trace
             ServerlessTraceAgent
             (full trace pipeline, sync flush)
```

At the end of every invocation (or on a timeout), each sub-agent is asked to
flush via `FlushableAgent.Flush()`. All three sub-agents use synchronous
flushing paths to ensure data is not lost when the sandbox is frozen.

## Key elements

### Root package (`pkg/serverless`)

| Element | Description |
|---|---|
| `FlushableAgent` interface | Single-method interface (`Flush()`) implemented by all three sub-agents. Enables generic flush loops in the extension code. |

### `pkg/serverless/metrics`

| Element | Description |
|---|---|
| `ServerlessMetricAgent` struct | Wraps a `dogstatsdServer.ServerlessDogstatsd` instance and an `aggregator.Demultiplexer`. Created by `Start(...)` and flushed with `Flush()`. |
| `MultipleEndpointConfig` interface | Abstraction over the config package. `MetricConfig` is the production implementation. |
| `DogStatsDFactory` interface | Abstraction for DogStatsD server creation. `MetricDogStatsD` is the production implementation. |
| `(c *ServerlessMetricAgent) Start(...)` | Builds the demultiplexer, starts the DogStatsD server. Sets `dogstatsd_packet_buffer_flush_timeout` to 1 ms to prevent buffered packets from being lost. |
| `(c *ServerlessMetricAgent) Flush()` | Calls `ServerlessFlush` with a sketch bucket offset so that distribution metrics land in the correct time bucket. |
| `(c *ServerlessMetricAgent) WaitForPendingSamples()` | Spins until the aggregator has consumed all buffered samples. Call this during shutdown before stopping. |
| `SketchesBucketOffset` field | `time.Duration` added to the current timestamp when computing the bucket for distribution/sketch metrics. Compensates for invocation start-time offset. |

### `pkg/serverless/trace`

| Element | Description |
|---|---|
| `ServerlessTraceAgent` interface | `Stop()`, `Flush()`, `Process(*api.Payload)`, `SetTags(map[string]string)`, `SetTargetTPS(float64)`, `SetSpanModifier(...)`, `GetSpanModifier()`. |
| `StartServerlessTraceAgent(args)` | Top-level constructor. Returns a `noopTraceAgent` when `args.Enabled` is false. Sets `serverless.enabled=true` in config (skips hostname resolution to save >1 s at startup) and disables `remote_configuration.apm_sampling`. |
| `StartServerlessTraceAgentArgs` struct | Controls trace-agent startup: `Enabled`, `LoadConfig`, `AdditionalProfileTags`, `FunctionTags`, and an optional `RCService` for remote config. |
| `Load` interface / `LoadConfig` struct | Abstraction for loading `*config.AgentConfig` from a file path; allows injection of a test double. |
| `filterSpan` function | Pre-configured `DiscardSpan` hook that removes spans generated by internal infrastructure: HTTP calls to `127.0.0.1:8124` (agent) and `:8125` (StatsD), and DNS lookups to localhost / non-routable addresses. |
| `spanModifier` struct | Implements `agent.SpanModifier`. Stamps `_dd.origin` on every span that does not already carry it. The origin value is derived from the cloud service type at runtime. |
| `disableTraceStatsEnvVar` (`DD_SERVERLESS_INIT_DISABLE_TRACE_STATS`) | When set to `"true"`, replaces the `Concentrator` with a no-op to skip trace stats computation in `serverless-init` deployments. |

### `pkg/serverless/trace/modifier`

Contains `TracerPayloadModifier` (consumed via `serverlessmodifier.NewTracerPayloadModifier`). Injects function-level tags into every `TracerPayload` arriving through the trace pipeline. This is set as `ta.TracerPayloadModifier` in `StartServerlessTraceAgent`.

### `pkg/serverless/logs`

| Element | Description |
|---|---|
| `SetupLogAgent(logChannel, sourceName, source, ...)` | Creates and starts a `ServerlessLogsAgent` backed by a `channel.Scheduler`. The scheduler feeds log messages received on `logChannel` into the standard logs pipeline. |
| `SetLogsTags(tags []string)` | Updates the tag set attached to all log messages without restarting the agent. Called when function tags are resolved after the first invocation. |

### `pkg/serverless/otlp`

| Element | Description |
|---|---|
| `ServerlessOTLPAgent` struct | Wraps a `coreOtlp.Pipeline`. Activated only when built with the `otlp` build tag. |
| `IsEnabled()` | Returns true if the OTLP endpoint is configured. |
| `NewServerlessOTLPAgent(serializer, tagger)` | Creates the OTLP pipeline. Returns `nil` on error. |
| `(o) Start()` / `(o) Stop()` | Lifecycle methods. `Stop()` waits up to 1 second for the collector to reach `Closed` state. |

Build tag: `//go:build otlp`. When the tag is absent, `otlp_no_otlp.go` provides stub implementations so callers compile without the OTel collector dependency.

### `pkg/serverless/tags`

| Element | Description |
|---|---|
| `ArrayToMap([]string) map[string]string` | Converts `"key:value"` tag strings (comma-separated or slice entries) to a map. Used to merge function tags from multiple sources. |
| `MapToArray(map[string]string) []string` | Reverse of `ArrayToMap`. |
| `MergeWithOverwrite(tags, overwritingTags)` | Returns a new map with `overwritingTags` taking precedence over `tags`. |
| `GetExtensionVersion() string` | Returns the extension version string injected at build time by the `datadog-lambda-extension` build scripts. Defaults to `"xxx"` if not replaced. |
| `ComputeStatsKey` / `ComputeStatsValue` | Tag pair (`_dd.compute_stats` = `"1"`) used to signal that trace stats should be computed for a given trace. |

### `pkg/serverless/env`

| Element | Description |
|---|---|
| `AzureAppServicesEnvVar` (`DD_AZURE_APP_SERVICES`) | Environment variable set to `"1"` by the Datadog Azure App Services extension. |
| `IsAzureAppServicesExtension() bool` | Returns true when running inside the Azure App Services extension context. |

### `pkg/serverless/streamlogs`

| Element | Description |
|---|---|
| `Formatter` struct | Implements the logs `Formatter` interface. Formats diagnostic log messages with metadata fields (integration name, type, status, timestamp, service, source, tags, message) as a single human-readable line. |

## Usage

### How it is used in the codebase

The primary consumers are:

- **`cmd/serverless-init/main.go`** — the `serverless-init` binary used in
  container-based serverless runtimes (Cloud Run, App Service, Container Apps).
  It calls `StartServerlessTraceAgent`, `ServerlessMetricAgent.Start`, and
  `SetupLogAgent` directly, then wraps the user's process and flushes on exit.

- **`cmd/trace-agent/subcommands/run/command.go`** — imports
  `pkg/serverless/tags` to read the extension version at startup.

- **`comp/trace/payload-modifier/impl/payloadmodifier.go`** — uses
  `pkg/serverless/trace/modifier` to inject function tags into trace payloads.

### Starting sub-agents (serverless-init pattern)

```go
import (
    "github.com/DataDog/datadog-agent/pkg/serverless/metrics"
    "github.com/DataDog/datadog-agent/pkg/serverless/trace"
    "github.com/DataDog/datadog-agent/pkg/serverless/logs"
)

// Metrics
metricAgent := &metrics.ServerlessMetricAgent{Tagger: tagger}
metricAgent.Start(5*time.Second, &metrics.MetricConfig{}, &metrics.MetricDogStatsD{}, false, extraTags)

// Traces
traceAgent := trace.StartServerlessTraceAgent(trace.StartServerlessTraceAgentArgs{
    Enabled:    true,
    LoadConfig: &trace.LoadConfig{Path: "datadog.yaml", Tagger: tagger},
})

// Logs
logChan := make(chan *config.ChannelMessage, 100)
logsAgent, err := logs.SetupLogAgent(logChan, "serverless", "serverless", tagger, compression, hostname)

// At end of invocation:
metricAgent.Flush()
traceAgent.Flush()
```

### Flush lifecycle

All three sub-agents should be flushed in order at the end of each invocation.
The trace agent's `Flush()` calls `FlushSync()` on the underlying `pkg/trace`
agent, which drains the in-flight trace pipeline synchronously. The metric
agent's `Flush()` calls `WaitForPendingSamples()` internally before
serializing. Do **not** call `Stop()` between invocations — only flush.

### Updating tags at runtime

Function-level tags (e.g. `aws_account`, `functionname`) are typically
resolved asynchronously after the first invocation. Push updated tags to the
running sub-agents without restarting them:

```go
tagMap := tags.ArrayToMap(resolvedTagArray)
traceAgent.SetTags(tagMap)
logs.SetLogsTags(tags.MapToArray(tagMap))
```

### How `pkg/serverless/logs` relates to `pkg/logs`

`SetupLogAgent` creates a `channel.Scheduler` from `pkg/logs/schedulers/channel`, which feeds a standard `*message.Message` pipeline (processor → strategy → sender → destination). The encoder used is `processor.JSONServerlessInitEncoder` (see [`pkg/logs`](logs/logs.md) pipeline overview). Unlike a normal agent deployment, the serverless logs agent does not use Autodiscovery or file tailing: all log messages are submitted directly over the `logChannel`.

### How `pkg/serverless/trace` relates to `pkg/trace`

`StartServerlessTraceAgent` calls `pkgtrace.NewAgent(ctx, conf, ...)` from `pkg/trace` (see [`pkg/trace`](trace/trace.md)) and disables the RC APM sampling product (`remote_configuration.apm_sampling = false`). It also sets `serverless.enabled = true` to skip hostname resolution. The `filterSpan` hook is registered as `agent.DiscardSpan` to drop internal infrastructure spans before any sampler runs. `Flush()` calls `FlushSync()` on the underlying `TraceWriter`, draining the pipeline synchronously — the same writer that normally uses a 5 s timer is drained on every Lambda invocation boundary.

### How `ServerlessMetricAgent` relates to `pkg/metrics`

`ServerlessMetricAgent` wraps a `dogstatsdServer.ServerlessDogstatsd` and an `aggregator.Demultiplexer`. Metrics received over UDP port 8125 become `MetricSample` values (from `pkg/metrics`) that are aggregated by `ContextMetrics` inside the demultiplexer. `ServerlessFlush` serialises the resulting `[]*Serie` and `SketchSeriesList` (see [`pkg/metrics`](metrics/metrics.md)) and sends them synchronously. The `SketchesBucketOffset` field compensates for invocation start-time drift by shifting the sketch bucket timestamp.

### OTLP in serverless (`pkg/serverless/otlp`)

`ServerlessOTLPAgent` wraps a `coreOtlp.Pipeline` (from `comp/otelcol/otlp`). When activated, it receives OTLP signals from the function, translates them via `pkg/opentelemetry-mapping-go` (see [`pkg/opentelemetry-mapping-go`](opentelemetry-mapping-go.md)), and forwards the resulting Datadog-format metrics and traces through the same demultiplexer and trace writer used by the non-OTLP paths. This requires the `otlp` build tag; without it the stubs in `otlp_no_otlp.go` compile in instead.

### Data flow summary (all signal types)

```
Lambda function
    │
    ├── UDP :8125 (DogStatsD) ──► ServerlessMetricAgent
    │                               └─ aggregator.Demultiplexer
    │                                     └─ pkg/metrics.ContextMetrics → Serie/SketchSeries
    │                                            └─ synchronous flush to Datadog intake
    │
    ├── HTTP :8126 (APM SDK)  ──► ServerlessTraceAgent (pkg/trace.Agent)
    │                               └─ filterSpan removes infra spans
    │                               └─ SpanModifier stamps _dd.origin
    │                               └─ TraceWriter.FlushSync() at invocation end
    │
    ├── Lambda Logs API       ──► logs.SetupLogAgent (channel.Scheduler)
    │                               └─ pkg/logs pipeline (JSON encoder, HTTP destination)
    │
    └── OTLP gRPC (optional)  ──► ServerlessOTLPAgent (pkg/opentelemetry-mapping-go)
                                    └─ routes translated metrics/traces to the pipelines above
```
