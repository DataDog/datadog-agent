## Top CPU-Consuming Functions (identified via code analysis)

Based on benchmark profiling and code inspection of comp/dogstatsd/server/:

1. **parseTags() in parse.go**
   - Called on every metric sample
   - Uses bytes.Split() which allocates intermediate [][]byte slice
   - Creates allocation pressure on hot path

2. **parseMetricSample() in parse.go**
   - Uses bytes.SplitN() for optional field parsing (|@, |#, |T, |c:, |e:, |card:)
   - Multiple allocations per packet for field splitting
   - Called for every incoming metric

3. **parseMetricSampleMetricType() in parse.go**
   - Uses sequential bytes.Equal() comparisons for type detection
   - Could be replaced with byte-switch for O(1) dispatch

4. **extractTagsMetadata() in enrich.go**
   - Scans full tag list even when all metadata fields already found
   - No early-exit optimization

5. **stringInterner.LoadOrStore() in intern.go**
   - Full cache reset eviction strategy (evicts hot strings)
   - High re-interning cost after cache fills
