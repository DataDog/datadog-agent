# dda inv test → bzl test Migration Plan

Make `dda inv test` default to `bzl test` for all packages with Bazel coverage, replacing
`go test` as the primary unit-test runner. `go test` remains as a silent fallback only for
the ~26 packages not yet covered by Bazel.

---

## Context and current state (2026-06-10)

### What already exists
- **937 `go_test` targets** across `pkg/` and `comp/` generated by Gazelle
- **`tasks/gotest.py`** already has the full Bazel test infrastructure:
  - `get_bazel_test_targets()` (line 171): runs `bazel cquery kind(go_test, scope)` to find covered targets
  - `_run_bazel_tests()` (line 255): runs `bazel test` on the discovered targets
  - `_BAZEL_QUERYABLE_ROOTS = ('//cmd/', '//comp/', '//pkg/')` (line 205)
  - `skip_tests_covered_by_bazel` (line 566, default `False`): skips Bazel-covered packages from `go test`
  - `run_bazel_tests` (line 568, default `False`): runs discovered targets via `bzl test`
  - `EXPERIMENTAL_USE_BAZEL_TESTS` env var (lines 588–591): back-door that flips both to `True`
- **Escape hatch**: Invoke auto-generates `--no-run-bazel-tests` / `--no-skip-tests-covered-by-bazel`
  from the boolean defaults

### What's missing / the gap
- **The cquery does not exclude `manual`-tagged targets** (line 213). When a privileged/eBPF
  `go_test` target is tagged `manual`, `cquery kind(go_test, ...)` still returns it, and
  `_run_bazel_tests()` then invokes it explicitly — bypassing `manual` semantics and running
  the test in an unprivileged sandbox where it fails. **This is the linchpin fix.**
- **Privileged/eBPF `go_test` targets are not yet tagged `manual`**. They would be picked
  up by the cquery and fail in the unprivileged sandbox.
- **The defaults have not been flipped** (`run_bazel_tests=False`, `skip_tests_covered_by_bazel=False`).

### Corrections to common misconceptions
1. **No `cmd/` test target generation needed.** All `cmd/agent` test files (36 total) live in
   subpackages (`cmd/agent/common/`, `cmd/agent/subcommands/*`) that already have Gazelle-generated
   `go_test` targets. The top-level `cmd/agent/BUILD.bazel` correctly has no `go_test` (there are
   no test files at the top level). Same for all other cmd/ binaries.
2. **The privileged-test classifier is `TEST_PACKAGES_LIST` in `tasks/system_probe.py:50–63`**
   (the KMT package set), not the `linux_bpf` build tag. Some packages with `linux_bpf` (e.g.
   config/DWARF-parsing tests) run fine in the unprivileged sandbox. Only sandbox-test each
   candidate to confirm it fails before tagging it `manual`.
3. **KMT jobs are completely separate.** `tasks/system_probe.py::test()` runs on dedicated
   kernel runners via `.gitlab/test/kernel_matrix_testing/`; it does not go through `dda inv test`.
   Tagging those targets `manual` removes them from the default `bzl test` path but does not affect
   the KMT path at all.

## Workstream A — Fix the cquery to exclude `manual` targets (CRITICAL, do first)

**Blocks everything else.** Without this fix, flipping the defaults immediately breaks CI.

### File: `tasks/gotest.py`

**Change 1 — line 213**: add `except attr(tags, ...)` to the cquery:
```python
# BEFORE:
output = bazel(ctx, "cquery", *bazel_flags, f"kind(go_test, {scope})", ...)

# AFTER:
output = bazel(ctx, "cquery", *bazel_flags,
               f"kind(go_test, {scope}) except attr(tags, \"manual\", {scope})",
               ...)
```

**Change 2 — line 275** (belt-and-suspenders): add `--test_tag_filters=-manual` to `base_args`
in `_run_bazel_tests()`:
```python
# BEFORE:
base_args = ["test", "--keep_going"]

# AFTER:
base_args = ["test", "--keep_going", "--test_tag_filters=-manual"]
```
This ensures even if a manual target somehow enters the list it won't execute.

### Verification
```sh
# Confirm manual targets are excluded by the new cquery:
bzl cquery 'kind(go_test, //pkg/ebpf/...) except attr(tags, "manual", //pkg/ebpf/...)' 2>&1 | head -20
# Confirm non-manual targets in the same tree are still returned.
```

**Effort: ~1 hour. On the critical path.**

## Workstream B — Tag privileged/eBPF `go_test` targets `manual`

Must land **with or before** the defaults flip (Workstream C).

### Identify which targets need `manual`

Start from the authoritative privileged-test set:
```python
# tasks/system_probe.py lines 50–63: TEST_PACKAGES_LIST
# These are the package trees KMT runs on dedicated kernel runners.
```

Query for go_test targets in those trees that are NOT already manual:
```sh
bzl cquery 'kind(go_test, //pkg/ebpf/... + //pkg/network/... + //pkg/collector/corechecks/ebpf/...
  + //pkg/discovery/module/... + //pkg/process/monitor/... + //pkg/dyninst/...
  + //pkg/gpu/ebpf/... + //pkg/security/ebpf/...) except attr(tags, "manual", //...)' 2>&1
```

**Before tagging any target `manual`**, confirm it actually fails in the unprivileged sandbox:
```sh
bzl test //pkg/ebpf:ebpf_test --platforms=//bazel/platforms:linux_arm64
# If it fails with a permissions/eBPF error → tag manual.
# If it passes → do NOT tag manual (no coverage drop needed).
```

### How to author the `manual` tag

**Do NOT re-run Gazelle** — it has no signal to emit `manual` tags and risks reformatting
`gotags`/`srcs` in the privileged BUILD files.

**Pattern** (verified from `pkg/ebpf/ebpftest/BUILD.bazel:23`):
```python
go_test(
    name = "foo_test",
    ...
    tags = ["manual"],  # keep
    target_compatible_with = ["@platforms//os:linux"],
)
```

The `# keep` comment prevents buildozer and future Gazelle runs from removing the manual tag.

**Bulk application**: use `buildozer` to add the tag, then hand-add `# keep`:
```sh
buildozer 'add tags manual' //pkg/ebpf:ebpf_test  # repeat per target
bzl run //bazel/buildifier -- pkg/ebpf/BUILD.bazel  # format
```

### Self-consistency: what happens to these packages in `go test`?

Once a target is tagged `manual`, `get_bazel_test_targets()` no longer returns it (after
Workstream A's fix). It is therefore NOT added to `exclude_packages`, so `go test` still runs it
via `test_flavor()`. Under unit-test tags (no `linux_bpf`), the package's `//go:build linux_bpf`
files compile away and `go test` sees an empty package — no test runs, no failure. Exactly
today's behavior. No coverage is silently dropped.

**Effort: ~0.5–1 day. On the critical path.**

## Workstream C — Flip the defaults (the "done" line)

Depends on A and B having landed, and E (differential verification) passing.

### File: `tasks/gotest.py`

**Change 1 — lines 566–568**: flip the defaults:
```python
# BEFORE:
skip_tests_covered_by_bazel=False,
write_bazel_test_list=None,
run_bazel_tests=False,

# AFTER:
skip_tests_covered_by_bazel=True,
write_bazel_test_list=None,
run_bazel_tests=True,
```

**Change 2 — lines 588–591**: resolve the `EXPERIMENTAL_USE_BAZEL_TESTS` block. With the
defaults now `True`, the env var's original meaning inverts. Options:
- **Recommended**: repurpose it as a legacy kill-switch: when set, force
  `skip_tests_covered_by_bazel=False, run_bazel_tests=False` (i.e. revert to pure `go test`).
  This gives an emergency CI escape hatch without needing a code deploy.
- Alternative: remove the block entirely; the `--no-run-bazel-tests` flag serves as the escape hatch.

**The escape hatch** (Invoke auto-generates this from the boolean default):
```sh
dda inv test --no-run-bazel-tests --no-skip-tests-covered-by-bazel  # full legacy go test
```

**Effort: ~1 hour. Final step.**

## Workstream D — CI confirmation

No edits required for most jobs; read-only verification.

### `.gitlab/build/source_test/linux.yml`
The `dda inv -- -e test $FLAVORS ...` job auto-flips to `bzl test` once the defaults change.
No edit required. Consider adding `--no-run-bazel-tests` temporarily here during a staged
rollout to decouple the code change from CI behaviour change, then remove it once verified.

### KMT jobs (`.gitlab/test/kernel_matrix_testing/`)
These call `tasks/system_probe.py::test()` directly, NOT `dda inv test`. Completely unaffected
by the defaults flip. Tagging privileged targets `manual` does not touch this path.

### `.gitlab/build/bazel/test.yml` (pre-existing `bazel test //...`)
This job already runs `bazel test --keep_going //...`; `//...` excludes `manual` targets via
wildcard semantics. After the flip, this job and the source_test job both cover the non-manual
set. Overlap is intentional: the bazel:test job is the broad platform guardrail; source_test is
the per-PR scoped/coverage run. Document this rather than eliminating either job.

**Effort: ~2–4 hours (reading + verification, minimal edits). Lands alongside C.**

## Workstream E — Differential verification (gates the flip)

Run this on a branch with A+B applied, before merging C.

### Step 1: Dump the new Bazel run set
```sh
dda inv test --write-bazel-test-list=/tmp/bazel_set.txt
```
This uses the existing `write_bazel_test_list` mechanism (gotest.py line 567, 699–702) to produce
a sorted list of `go_test` labels the new path will run.

### Step 2: Dump the legacy go-test package set
```sh
dda inv test --no-run-bazel-tests --no-skip-tests-covered-by-bazel --module=. 2>&1 \
  | grep "^=== RUN\|testing" | head -100
```
Or extract the package list from `go list -tags "..." ./...`.

### Step 3: Cross-check for silent drops
Map Bazel labels to import paths (label `//pkg/x:y_test` → `github.com/.../pkg/x`) and diff.
The expected delta between "legacy go test packages" and "new bzl test labels" is:
- The `manual`-tagged privileged set (from `TEST_PACKAGES_LIST`)
- The ~26 packages in `root BUILD.bazel`'s `# gazelle:exclude` list that have no `go_test` target

Any package outside that delta that appears in neither set is a **silent coverage drop** —
investigate before proceeding.

### Step 4: Pass/fail parity
Run the full test suite both ways on a CI branch (the `TRIGGER_ALL_TESTS_PATHS` guard in
gotest.py:72 includes `tasks/gotest.py` changes, forcing a full test run). Confirm identical
pass/fail counts modulo the known delta. The combined summary block (gotest.py lines 764–780)
reports total/passed/failed for cross-checking.

### Step 5: Per-manual-tagged target sandbox check
For every target newly tagged `manual`, confirm:
- It fails in the sandbox *before* tagging (justifying the tag).
- A non-manual sibling in the same package tree still runs and passes.

**Effort: ~1 day. Blocks Workstream C.**
